gowasm

Browser shell for the Rust-powered Go runtime. The real worker/module boundary is now exposed in the shell too: edit a multi-file workspace, point the shell at package-test or run-entry files, and load remote module bundles into the same virtual __module_cache__/... view the worker harness already exercises.

Project Archive Import

Archive import replaces the editable workspace with validated UTF-8 text files, rejects traversal paths and reserved __module_cache__/... entries, and strips a detected wrapper or single-module root before opening the project.

Boot URL

Boot-manifest fetches stay browser-owned. A remote manifest only auto-loads when the URL explicitly includes consent; otherwise the shell waits for this button.

Workspace Snapshot

Snapshots serialize the editable workspace, configured module roots, selected targets, loaded-module references, and forward-compatible settings/example placeholders into a stable JSON file.

Imported archives, boot-loaded workspaces, and snapshot restores keep one latest imported-workspace cache entry. Remote module bundles stay worker-owned in IndexedDB and this panel only reports their status and clear actions.

Run and test requests automatically combine the editable workspace with any loaded remote module bundles. Format and lint stay scoped to the editable workspace files so cached remote sources remain host-owned: __module_cache__/... paths are reserved for projected bundles, and failed reloads leave prior bundles marked stale until the next successful refresh. Package tests still target same-package top-level Test* functions; package main is now supported through the generated runner, while external _test packages remain out of scope.
